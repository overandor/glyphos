# Skills Integration

Integration of Membra ChatGPT Export Skills with the codebase.

## Components

### Underwriting Pipeline
Implements the full underwriting pipeline: `intake → collect → value → benchmark → risk → eligibility → decide → memo`

**File:** `underwriting_pipeline.py`

**Features:**
- GitHub activity scoring
- Financial metrics (ARR/MRR, IP assignment)
- Compute memory continuity scoring
- Risk grade assignment (A-E)
- Borrowing base calculation
- Condition and covenant generation

**Example:**
```python
from skills_integration import UnderwritingPipeline, UnderwritingInput

pipeline = UnderwritingPipeline()
input_data = UnderwritingInput(
    repo_count=5,
    test_coverage=0.3,
    arr_mrr=100000,
    ip_assigned=True,
    memory_state_gb=16,
    recoverable_state_score=0.4
)
output = pipeline.run(input_data)
print(f"Risk Grade: {output.risk_grade}")
print(f"Borrowing Base: ${output.borrowing_base:,.2f}")
```

### Compute Capital Valuation
Values machines as: `hardware + recoverable utility - reconstruction cost`

**File:** `compute_capital_valuation.py`

**Features:**
- Hardware depreciation calculation
- Recoverable state utility scoring
- Reconstruction cost estimation
- Productivity multiplier calculation
- Appraisal memo generation

**Thesis:** A machine with accumulated productive state is worth more than resale value because it reduces future compute requirements.

**Example:**
```python
from skills_integration import ComputeCapitalValuation, MachineState

valuer = ComputeCapitalValuation()
state = MachineState(
    hardware_value_usd=3000,
    ram_gb=64,
    age_years=3,
    cache_hit_rate=0.75,
    jobs_completed=2500,
    restore_success_rate=0.92
)
result = valuer.value(state)
print(f"Total Value: ${result.total_value:,.2f}")
print(f"Productivity Multiplier: {result.productivity_multiplier:.2f}x")
```

### ProofBook Integration
Plans and implements auditable evidence storage for underwriting decisions.

**File:** `proofbook_integration.py`

**Features:**
- SHA-256 content hashing
- Proof chain creation and verification
- Chain integrity validation
- Integration planning for underwriting, receipts, and valuation

**Sequence:**
1. Hash and record underwriting memos
2. Attach signed reuse receipts from memory or compute runs
3. Feed receipts into node valuation model
4. Rerun underwriting pipeline with real proof, not claims

**Example:**
```python
from skills_integration import ProofBookIntegration
from pathlib import Path

proofbook = ProofBookIntegration(Path("proofbook_storage"))
chain = proofbook.start_chain(
    "underwriting-20240101",
    underwriting_decision,
    "underwriting_decision",
    "underwriting_pipeline",
    metadata
)
print(f"Chain ID: {chain.chain_id}")
print(f"Root Hash: {chain.root_hash}")
```

## Usage Pattern

The complete workflow:

```python
import asyncio
from pathlib import Path
from layer_crawler_etl import ETLPipeline, Source, SubjectType
from layer_crawler_etl import CodeCrawler, BrowserRuntimeCrawler
from skills_integration import UnderwritingPipeline, ComputeCapitalValuation, ProofBookIntegration

# 1. Crawl and generate receipts
pipeline = ETLPipeline(Path("receipts"))
pipeline.register_crawler(SubjectType.CODE, CodeCrawler())
pipeline.register_crawler(SubjectType.BROWSER_RUNTIME, BrowserRuntimeCrawler())
receipt = await pipeline.run([Source(type="repo", location=".")], "My System")

# 2. Run underwriting with evidence
underwriting = UnderwritingPipeline()
underwriting_output = underwriting.run(input_data)

# 3. Value the machine
valuation = ComputeCapitalValuation()
valuation_result = valuation.value(machine_state)

# 4. Store in ProofBook for auditability
proofbook = ProofBookIntegration(Path("proofbook_storage"))
chain = proofbook.start_chain("underwriting-chain", underwriting_output, ...)
proofbook.append_to_chain(chain.chain_id, receipt, ...)
proofbook.append_to_chain(chain.chain_id, valuation_result, ...)
```
